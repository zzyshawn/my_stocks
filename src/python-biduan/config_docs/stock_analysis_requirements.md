# 股票分析系统需求文档

## 1. 系统概述

### 1.1 系统目标
本系统基于缠论理论，对股票数据进行自动化分析，识别K线形态、分型、笔、线段、中枢等关键结构，为投资决策提供技术分析支持。

### 1.2 核心功能流程
**强制执行流程顺序：**
1. 先检测K线状态，进行K线合并
2. 生成顶分型和底分型
3. 生成有效分型
4. 生成第一次的顶底
5. 检测顶底之间的是否还有其他顶底（这个就是笔）
6. 笔的定义与实现
7. 生成笔中枢

### 1.3 数据产生和使用原则
- **强制执行要求：所有计算数据必须由核心代码产生，导出Excel和输出HTML必须使用核心计算数据的结果**
- **具体实现要求：**
  - 核心代码（analyzer.py）负责所有缠论分析计算
  - 导出代码（excel_export.py）必须直接使用核心代码的计算结果，先输出
  - HTML输出代码必须直接使用核心代码的计算结果，后输出
  - 禁止在导出代码中重新计算核心分析数据
  - 确保数据一致性和系统可维护性

## 3. 核心功能流程详细需求

### 3.1 第一步：K线状态检测与合并

#### 3.1.1 操作流程
- 读入K线序列后，先处理K线的包含状态，引入合并K线序列
- 对K线进行状态标记：上升、下降、上升包含、下降包含
- 全部完成后，将标记打到原始K线上

#### 3.1.2 状态定义

- **上升**：当日高点 > 前日高点 且 当日低点 > 前日低点，将标记到合并K线序列
- **下降**：当日高点 < 前日高点 且 当日低点 < 前日低点，将标记到合并K线序列
- **上升包含**：当日高点 >= 前日高点 且 当日低点 <= 前日低点（前日K线被今日K线包含） 或 当日高点 <= 前日高点 且 当日低点 >= 前日低点（今日K线被前日K线包含），判断是上升包含还是下降包含通过前日合并后的K线状态来操作。如果是前日合并后的K线为上升状态，则是上升包含；如果是前日合并后的K线为下降状态，则是下降包含。上升包含的合并K线高点=所有K线的最高点，上升包含的合并K线低点=所有K线的最低点中的最高点，保留新合并K线，删除对应旧K线
- **下降包含**：同上升包含，根据前日合并后的K线状态来判断具体是上升包含还是下降包含。如果是前日合并后的K线为上升状态，则是上升包含；如果是前日合并后的K线为下降状态，则是下降包含。下降包含的合并K线高点=所有K线的最高点的最低点，下降包含的合并K线低点=所有K线的最低点

#### 3.1.3 输出要求
- 输出测试用Excel，新增"K线状态"列
- 标记上升、下降、上升包含、下降包含
- 将上升包含和对应的被包含K线行底色标为黄色，下降包含和被包含K线行底色标为绿色
- 新增"合并K线状态"列：在Excel输出中新增一列显示合并后的K线状态，合并后的K线状态标记为：上升、下降。合并K线逻辑参考3.1.2的上升包含和下降包含中的定义，经过包含关系处理后，最终形成的合并K线状态只标记为"上升"或"下降"，不再包含"上升包含"或"下降包含"状态
- 新增"包含起始日期"列：在Excel输出中新增一列显示包含关系的起始日期，对于包含关系的K线，填写该包含关系的起始日期；对于非包含关系的K线，该列为空

#### 3.1.4 HTML图表包含关系标记
- 在HTML图表中，对包含关系的K线进行颜色标记
- 上升包含：使用黄色背景标记，覆盖合并K线的最高值和最低值范围
- 下降包含：使用绿色背景标记，覆盖合并K线的最高值和最低值范围
- 标记范围：对于合并的多个K线，颜色标记应覆盖这些K线的最高价和最低价之间的范围
- 颜色透明度：使用适当透明度（0.4），确保不影响K线本身显示

### 3.2 第二步：生成顶分型和底分型

#### 3.2.1 分型定义
- 分型识别应基于合并后的K线序列进行标记
- 顶分型：在合并K线序列中，当第i根K线的高点同时高于第i-1和第i+1根K线的高点时，构成顶分型
- 底分型：在合并K线序列中，当第i根K线的低点同时低于第i-1和第i+1根K线的低点时，构成底分型
- 分型标记：识别出的分型应映射回原始K线序列中对应的位置进行标记

#### 3.2.2 最后一个K线的分型处理
- 当处理完所有K线分型后，如果最后一个合并K线处于上升状态，则该K线的最高点视为潜在顶分型
- 当处理完所有K线分型后，如果最后一个合并K线处于下降状态，则该K线的最低点视为潜在底分型
- 最后一个K线的分型不一定会形成标准分型，因为它缺少后续K线的验证
- 潜在顶分型和底分型就当作顶分型和底分型来处理
- 在图表中可以用特殊标记（如半透明的标记）来表示最后一个K线的潜在分型

#### 3.2.3 HTML图表分型标记
- 在HTML图表中，对顶分型和底分型进行标记
- 顶分型：使用红色三角形标记，在对应K线的最高点位置显示
- 底分型：使用蓝色三角形标记，在对应K线的最低点位置显示
- 标记位置：对于包含关系处理后的分型，应映射回原始K线的位置进行标记
- 标记大小：使用合适的标记大小（如15像素），确保清晰可见但不影响图表整体观感

### 3.3 第三步：生成有效分型

#### 3.3.1 有效分型定义
- **顶顶分型**：当前顶分型的最高值>=前顶分型的最高值 且 当前顶分型的最高值>后一个顶分型的最高值
- **底底分型**：当前底分型的最低值<=前底分型的最低值 且 当前底分型的最低值<后一个底分型的最低值

#### 3.3.2 有效分型组合定义
- **有效顶底组合**：前一个有效分型是顶顶分型，当前是底底分型
- **有效底顶组合**：前一个有效分型是底底分型，当前是顶顶分型
- **有效底底组合**：连续两个底底分型的有效组合
- **有效顶顶组合**：连续两个顶顶分型的有效组合

#### 3.3.3 输出要求
- 新增"有效分型"列：在Excel输出中新增一列标记有效分型，包括顶顶分型和底底分型
- 新增"有效分型组合"列：在Excel输出中新增一列标记有效分型组合，包括有效顶底组合、有效底顶组合、有效底底组合和有效顶顶组合
- 组合的完整定义：有效分型组合应包含完整的定义，包括：
  - 有效顶底组合：前一个有效分型是顶顶分型，当前是底底分型
  - 有效底顶组合：前一个有效分型是底底分型，当前是顶顶分型
  - 有效底底组合：前一个有效分型是底底分型，当前是底底分型
  - 有效顶顶组合：前一个有效分型是顶顶分型，当前是顶顶分型

#### 3.3.4 HTML图表有效分型标记
- 在HTML图表中，对顶顶分型和底底分型进行特殊标记
- 顶顶分型：使用橙色三角形标记，在对应K线的最高点位置显示
- 底底分型：使用绿色三角形标记，在对应K线的最低点位置显示

### 3.4 第四步：生成第一次的顶底

#### 3.4.1 顶底标记定义
- **上顶**：经过组合调整后确定的关键高点
- **下底**：经过组合调整后确定的关键低点

#### 3.4.2 基本标记规则
- 【强制规则】所有有效分型是顶顶分型，则顶底标注为上顶（不受间隔K线限制）
- 【强制规则】所有有效分型是底底分型，则顶底标注为下底（不受间隔K线限制）

#### 3.4.3 输出要求
- 新增"顶底"列：在Excel输出中新增一列，将组合调整中标记的上顶和下底填入，用于表示经过组合调整后确定的关键高低点

### 3.5 第五步：检测顶底之间的其他顶底

#### 3.5.1 组合调整规则

**有效顶顶组合处理：**
- 如果有效分型组合是有效顶顶组合，则在两个顶顶分型中寻找一个底分型
- 满足条件：这个底分型所在K线距离前一个顶顶分型最少间隔3个K线，后一个顶顶分型最少间隔3个K线
- 寻找优先级：寻找的底分型需要最低，如果不是最低则次低以此类推
- 标记方式：记录为下底，两个顶顶分型记录为上顶,如果无满足的，删除上顶低的那个的上顶标记

**有效底底组合处理：**
- 如果有效分型组合是有效底底组合，则在两个底底分型中寻找一个顶分型
- 满足条件：这个顶分型所在K线距离前一个底底分型最少间隔3个K线，后一个底底分型最少间隔3个K线
- 寻找优先级：寻找的顶分型需要最高，如果不是最高则次高以此类推
- 标记方式：记录为上顶，两个底底分型记录为下底，如果无满足的，删除下底高的那个的下底标记

**有效顶底组合处理：**
- 如果有效分型组合是有效顶底组合（前一个有效分型是顶顶分型，当前是底底分型），则在这两个有效分型之间寻找一个顶分型和一个底分型
- **处理规则：只标记第一对满足条件的顶分型和底分型**
- **寻找顺序：**
  1. 先从前一个上顶开始，寻找一个间隔大于3个K线的底分型
  2. 在找到的底分型之后，寻找一个间隔大于3个K线的顶分型
  3. 找到的这个顶分型距离有效底底组合间隔大于3个K线
- **寻找优先级：第一个满足条件的组合**
- 必要条件：需同时存在底分型和顶分型，且顶的最高点一定高于底的最低点
- 标记方式：标记这两个顶分型和底分型为上顶和下底，最后更新相关数据
- 原有标记：原有的顶顶分型为上顶，底底分型为下底，如果找到合格的中间顶分型和底分型，则中间顶分型记录为上顶，中间底分型记录为下底

**有效底顶组合处理：**
- 如果有效分型组合是有效底顶组合（前一个有效分型是底底分型，当前是顶顶分型），则在这两个有效分型之间寻找一个顶分型和一个底分型
- **处理规则：只标记第一对满足条件的顶分型和底分型**
- **寻找顺序：**
  1. 先从前一个下底开始，寻找一个间隔大于3个K线的顶分型
  2. 在找到的顶分型之后，寻找一个间隔大于3个K线的底分型
  3. 找到的这个底分型距离有效顶顶组合间隔大于3个K线
- **寻找优先级：第一个满足条件的组合**
- 必要条件：需同时存在底分型和顶分型，且顶的最高点一定高于底的最低点，
- 标记方式：原有的底底分型为下底，顶顶分型为上顶，如果找到合格的中间顶分型和底分型，则中间顶分型记录为上顶，中间底分型记录为下底，最后更新相关数据


#### 3.5.2 顶底交替规则
- 上顶和下底需要交替出现，且间隔的K线最小是3个

#### 3.5.3 顶底二次调整
  **当前上顶**
  则在这两个有效分型之间寻找一个顶分型和一个底分型
  **处理规则：只标记第一对满足条件的顶分型和底分型**
- **寻找顺序：**
  1. 先从前一个下底开始，寻找一个间隔大于3个K线的顶分型
  2. 在找到的顶分型之后，寻找一个间隔大于3个K线的底分型
  3. 找到的这个底分型距离有效顶顶组合间隔大于3个K线
- **寻找优先级：第一个满足条件的组合**
- 必要条件：需同时存在底分型和顶分型，且顶的最高点一定高于底的最低点，
- 标记方式：如果找到合格的中间顶分型和底分型，则中间顶分型记录为上顶，中间底分型记录为下底，最后更新相关数据

   **当前下底**
  则在这两个有效分型之间寻找一个顶分型和一个底分型
  **处理规则：只标记第一对满足条件的顶分型和底分型**
  - **寻找顺序：**
  1. 先从前一个上顶开始，寻找一个间隔大于3个K线的底分型
  2. 在找到的底分型之后，寻找一个间隔大于3个K线的顶分型
  3. 找到的这个顶分型距离有效底底组合间隔大于3个K线
- **寻找优先级：第一个满足条件的组合**
- 必要条件：需同时存在底分型和顶分型，且顶的最高点一定高于底的最低点
- 标记方式：标记这两个顶分型和底分型为上顶和下底，最后更新相关数据
- 原有标记：如果找到合格的中间顶分型和底分型，则中间顶分型记录为上顶，中间底分型记录为下底

### 3.6 第六步：笔的定义与实现

#### 3.6.1 笔的基本定义
- 笔由连接上顶的高点和下底的低点的线段构成
- 上行笔：当前是上顶，代表一笔的终点，用红虚线表示
- 下行笔：当前是下底，代表一笔的终点，用绿虚线表示

#### 3.6.2 笔的识别规则
- **强制执行要求：笔的识别必须在当下进行判断，只能使用当前和之前的信息，不能使用未来的信息**
- **具体实现要求：**
  - 只能向前搜索，不能向后搜索
  - 每个时间点只能使用到该时间点为止的已知信息
  - 笔的方向判断必须基于已知的前一个顶底分型和当前顶底分型
  - 禁止使用未来的顶底分型信息进行判断

#### 3.6.3 笔和K线索引对齐要求
- 每个顶底都是一笔
- 第一个顶底是一笔，但它没有笔开始的记录（只有结束）
- 第二个顶底是完整的一笔，因为有开始和结束
- 第三个顶底就可以绘制第一个中枢了
- 所有的顶底都可以认为是一笔的终点，但第一个顶点因为没有开始时间所以不用，实际返回的就是第二个点
- 但应该把第一个不完整的笔也返回，这样就不会错位了
- 笔的起始位置和K线起始位置必须正确对齐，确保数据一致性

#### 3.6.4 HTML图表笔的显示
- 在HTML图表中展示笔的连线，用一根线展示
- 生成一个按钮来控制笔连线的显示/隐藏

### 3.7 第七步：生成笔中枢

#### 3.7.1 笔中枢基本定义
- 中枢由连续三笔的重叠区间构成
- 笔的定义参照3.6.1
- 至少需要连续3笔形成笔中枢
- 上行中枢的上沿ZG为连续三笔中最高的低点
- 上行中枢的下沿ZD为连续三笔中最低的高点
- 下行中枢的上沿ZG为连续三笔中最高的低点
- 下行中枢的下沿ZD为连续三笔中最低的高点
- 上行中枢：由上行笔和下行笔交替构成，整体呈现向上趋势
- 下行中枢：由下行笔和上行笔交替构成，整体呈现向下趋势

#### 3.7.2 笔中枢的形成条件
- 新增中枢顶ZG，中枢底ZD，中枢笔计数
- 非顶底K线沿用前一个K线的ZG和ZD
- K线是上顶和下底时修改当前ZG和ZD值

#### 3.7.3 笔中枢的级别
- 一级中枢：由基本笔构成的中枢
- 二级中枢：由一级线段构成的中枢
- 三级中枢：由二级线段构成的中枢

#### 3.7.4 一级笔中枢的实现逻辑
- **强制执行要求：中枢的计算必须在当下进行判断，只能使用当前和之前的信息，不能使用未来的信息**
- **具体实现要求：**
  - 只能向前搜索，不能向后搜索
  - 每个时间点只能使用到该时间点为止的已知信息
  - ZG/ZD值的计算必须基于已知的前一个中枢和当下顶底分型
  - 禁止使用未来的顶底分型信息进行中枢扩展判断

##### 3.7.4.1 初始化前顶底
- 前顶底日期=当下这笔的起点日期

##### 3.7.4.2 初始化第一个中枢
- 从头开始查找，记录第一个顶底
- 如果是上顶则ZG=当前顶的高，ZG时间=当下时间
- 如果是下底ZD=当前底的低，ZD时间=当下时间

##### 3.7.4.3 第二个顶底开始
- 如果当前顶底是上顶，记录 ZG时间=当下时间，ZG=当前顶的高，ZD时间=前一个下底的时间，ZD=前一个下底的低
- 如果当前顶底是下底，记录 ZG时间=前一个上顶的时间，ZG=前一个上顶的高，ZD时间=当下时间，ZD=当前底的低

##### 3.7.4.4 正常情况顶底判断
对于第三个及以后的顶底，根据当前顶底相对于前一个中枢的位置进行扩展：

**当下K线是下底：**
- 如果在前一个K线中枢内（下底的低 >= 前ZD 且 下底的低 <= 前ZG），则ZG = 前一个K线记录的ZG，ZD = 当下K线的低
- 如果在前一个K线中枢下（下底的低 < 前ZD），则ZG = 前一个K线ZG，ZD = 前一个K线的ZD
- 如果在前一个K线中枢上（下底的低 > 前ZG），则ZG = 前一个上顶的高，ZD = 当下K线的低

**当下K线是上顶：**
- 如果在前一个K线中枢内（上顶的高 >= 前ZD 且 上顶的高 <= 前ZG)，则ZG = 当下K线的高，ZD = 前一个K线的ZD
- 如果在前一个K线中枢上（上顶的高 > 前ZG），则ZG = 前一个K线ZG，ZD = 前一个K线的ZD
- 如果在前一个K线中枢下（上顶的高 < 前ZD），则ZG = 当下K线ZG，ZD = 前一个下底的低

##### 3.7.4.5 非顶底K线的数据沿用规则
- 如果当下K线不是顶底分型，则沿用最近一个顶底分型的以下数据：
  - ZG值获取日期
  - ZD值获取日期
  - 中枢ZG值
  - 中枢ZD值
  - 中枢位置

#### 3.7.5 笔中枢的破坏
- 中枢破坏：走势突破中枢边界
- 破坏确认：需要连续两笔以上确认突破
- 中枢延伸：在原中枢区间内继续震荡

#### 3.7.6 笔中枢在图表中的显示
- 展示的是1级笔中枢，高低点使用ZG和ZD
- 绘制规则：当当前K线的ZG != 前一个K线的ZG 或 当前K线的ZD != 前一个K线的ZD时，开始绘制前一个中枢
- 起点：当前ZG或ZD发生变化时的前一个K线时间（即最初ZG或ZD开始保持不变的时间）
- 终点：当下时间（即ZG或ZD发生变化的时间）
- 使用矩形区域标记中枢区间
- 背景：透明
- 边框：黄色，线宽1像素
- 标注：显示中枢级别和序号（格式：1级-中枢N）
- 序号重置规则：当当前ZG和ZD同时发生变化时，下一个中枢序号重置为1

#### 3.7.7 输出要求
- 在Excel输出中新增"中枢ZG"列：显示中枢上沿ZG的数值
- 在Excel输出中新增"中枢ZD"列：显示中枢下沿ZD的数值
- 在Excel输出中新增"前顶底日期"列：显示当前笔所对应的前一个顶底的日期
- 在Excel输出中新增"ZG值获取日期"列：显示当前ZG值所对应的日期
- 在Excel输出中新增"ZD值获取日期"列：显示当前ZD值所对应的日期
- 日期格式为YYYY-MM-DD(日线) 或 (分钟)YYYY-MM-DD HH:MM:SS

## 7. 技术指标模块

### 7.1 RSI 指标系列

#### 7.1.1 RSI (相对强弱指数)
- **计算方式**：基于收盘价
- **默认周期**：6（与 target_list.xlsx 中的 RSI_6 对应）
- **函数位置**：`chanlun/utils/indicators.py::TechnicalIndicators.calculate_rsi()`

#### 7.1.2 RSI_H (最高价 RSI)
- **计算方式**：基于最高价
- **默认周期**：6
- **函数位置**：`chanlun/utils/indicators.py::TechnicalIndicators.calculate_rsi_h()`

#### 7.1.3 RSI_L (最低价 RSI)
- **计算方式**：基于最低价
- **默认周期**：6
- **函数位置**：`chanlun/utils/indicators.py::TechnicalIndicators.calculate_rsi_l()`

### 7.2 成交量指标

#### 7.2.1 基本成交量
- **数据来源**：K线数据的 volume 字段
- **单位**：手

#### 7.2.2 成交量均线
- **计算方式**：成交量的简单移动平均
- **默认周期**：10, 20（可配置）
- **函数位置**：`chanlun/utils/indicators.py::TechnicalIndicators.calculate_volume_ma()`

### 7.3 筹码分布

#### 7.3.1 计算原理
- **数据来源**：5分钟K线数据
- **聚合方式**：将5分钟K线的成交量按价格区间分配
- **日线筹码**：1天 = 48根5分钟K线
- **30分钟筹码**：1个30分钟 = 6根5分钟K线

#### 7.3.2 回看周期
- **默认周期**：1（可输入参数，最大100）
- **含义**：回看N个周期的5分钟数据计算筹码分布

#### 7.3.3 输出指标
| 指标 | 说明 |
|------|------|
| 平均成本 | 加权平均价格 |
| 当前价格 | 目标时间点的收盘价 |
| 获利比例 | 低于当前价格的筹码占比 |
| 集中度 | 1 - (90%筹码区间宽度 / 当前价格) |
| 90%筹码区间 | 90%筹码集中的价格区间 |
| 70%筹码区间 | 70%筹码集中的价格区间 |

#### 7.3.4 导出功能
- **函数位置**：`chanlun/utils/chip_export.py`
- **导出格式**：Excel (.xlsx)
- **输出目录**：`{股票代码}/chip/`
- **文件命名**：`{股票代码}_{周期类型}_chip_lookback{N}.xlsx`

#### 7.3.5 导出文件结构
**Sheet 1: 筹码分布**
| 价格区间 | 成交量 | 成交占比 | 累计占比 |
|----------|--------|----------|----------|

**Sheet 2: 统计指标**
| 指标 | 值 |
|------|-----|
| 平均成本 | - |
| 当前价格 | - |
| 获利比例 | -% |
| 集中度 | -% |
| 90%筹码区间 | 低 - 高 |
| 70%筹码区间 | 低 - 高 |

### 7.4 配置参数

```yaml
analysis:
  rsi:
    period: 6  # 默认周期
  
  volume_ma:
    periods: [10, 20]  # 可配置多个周期
  
  chip_distribution:
    default_lookback: 1  # 默认回看周期数
    max_lookback: 100  # 最大回看周期数
    price_bins: 50  # 价格区间数量
    periods: [1, 5, 10]  # 默认计算的回看周期列表

output:
  chip_distribution:
    dir: "chip"  # 子目录名称
    formats: ["xlsx"]  # 支持的导出格式
```
