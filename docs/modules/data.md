# 数据模块需求

## 1. 实时股票数据获取模块

### 1.1 功能描述
负责从多种数据源获取实时股票数据，包括行情数据、财务数据等。

### 1.2 功能需求

| 编号 | 需求项 | 优先级 | 描述 |
|------|--------|--------|------|
| R1.1.1 | 多数据源支持 | P0 | 支持主流数据源（如：akshare、tushare、新浪财经、东方财富等） |
| R1.1.2 | 实时行情数据 | P0 | 获取实时股价、成交量、涨跌幅等基本行情数据 |
| R1.1.3 | 历史K线数据 | P0 | 获取历史日K、周K、月K等K线数据 |
| R1.1.4 | 财务数据 | P1 | 获取财报数据、财务指标等 |
| R1.1.5 | 数据缓存 | P1 | 实现本地数据缓存，减少重复请求 |
| R1.1.6 | 数据源切换 | P2 | 支持自动切换数据源，保证数据获取稳定性 |
| R1.1.7 | 数据校验 | P1 | 对获取的数据进行有效性校验 |

### 1.3 数据字段要求

**基本行情数据：**
- 股票代码 (symbol)
- 股票名称 (name)
- 当前价格 (price)
- 开盘价 (open)
- 收盘价 (close)
- 最高价 (high)
- 最低价 (low)
- 成交量 (volume)
- 成交额 (amount)
- 涨跌幅 (change_pct)
- 涨跌额 (change)
- 换手率 (turnover_rate)
- 市盈率 (pe_ratio)
- 市净率 (pb_ratio)
- 更新时间 (update_time)

**K线数据：**
- 日期 (date)
- 开盘价 (open)
- 收盘价 (close)
- 最高价 (high)
- 最低价 (low)
- 成交量 (volume)
- 成交额 (amount)
- 涨跌幅 (change_pct)

## 2. 股票数据源

### 2.1 数据存放位置
- **基础路径**：`H:\股票信息\股票数据库\daban\股票数据`
- **子目录结构**：以股票代码命名的子文件夹，每个股票目录下包含历史数据和 realtime 文件夹
- **股票数据目录**：`H:\股票信息\股票数据库\daban\股票数据\{股票代码}`
- **实时数据目录**：`H:\股票信息\股票数据库\daban\股票数据\{股票代码}\realtime`
- **示例路径**：
  - `H:\股票信息\股票数据库\daban\股票数据\000001\000001_d.xlsx` (日线数据)
  - `H:\股票信息\股票数据库\daban\股票数据\000001\000001_30.xlsx` (30 分钟数据)
  - `H:\股票信息\股票数据库\daban\股票数据\000001\000001_5.xlsx` (5 分钟数据)
  - `H:\股票信息\股票数据库\daban\股票数据\000001\realtime\` (实时临时数据)

### 2.2 数据文件规范

**历史K线数据：**
- **文件格式**：Microsoft Excel 工作簿 (.xlsx)
- **数据类型**：
  - 日线数据：`{股票代码}_d.xlsx`
  - 30分钟线数据：`{股票代码}_30.xlsx`
  - 5分钟线数据：`{股票代码}_5.xlsx`
- **字段要求**：至少包含日期、开盘价、最高价、最低价、收盘价、成交量字段

**实时临时数据：**
- **存放位置**：`{股票代码}/realtime/` 文件夹
- **用途**：存放该股票的实时行情、临时分析结果等
- **特点**：数据会频繁更新，可定期清理

### 2.3 目录结构规范

```
H:/股票信息/股票数据库/daban/股票数据/
├── 000001/                    # 平安银行
│   ├── 000001_d.xlsx         # 日线数据
│   ├── 000001_30.xlsx        # 30分钟数据
│   ├── 000001_5.xlsx         # 5分钟数据
│   └── realtime/              # 实时临时数据
├── 600000/                    # 浦发银行
│   ├── 600000_d.xlsx
│   ├── 600000_30.xlsx
│   ├── 600000_5.xlsx
│   └── realtime/              # 实时临时数据
└── ...                        # 其他股票
```

**命名规则：**
- 历史数据子目录：以股票代码命名
- 数据文件：`{股票代码}_{周期}.xlsx`
- 实时数据目录：`realtime/`（位于股票代码目录下）
- 数据周期标识：
  - 日线：`_d` (如: `000001_d.xlsx`)
  - 30分钟线：`_30` (如: `000001_30.xlsx`)
  - 5分钟线：`_5` (如: `000001_5.xlsx`)

## 3. 历史数据合并模块

### 3.1 功能描述
提供股票历史数据的读取、合并和处理功能，支持多股票、多周期数据的合并分析。

### 3.2 功能需求

| 编号 | 需求项 | 优先级 | 描述 |
|------|--------|--------|------|
| R3.1 | 单股数据加载 | P0 | 加载单只股票指定周期的K线数据 |
| R3.2 | 多股数据加载 | P0 | 批量加载多只股票的K线数据 |
| R3.3 | 按日期合并 | P0 | 横向合并多只股票数据，用于比较分析 |
| R3.4 | 按股票合并 | P0 | 纵向合并多只股票数据，用于批量分析 |
| R3.5 | 多周期合并 | P1 | 合并同一股票不同周期的数据 |
| R3.6 | 日期对齐 | P1 | 支持不同股票数据的日期对齐 |
| R3.7 | 数据导出 | P1 | 支持导出为 Excel/CSV 格式 |

### 3.3 模块结构

```
src/data/
├── __init__.py
└── data_merger.py       # 数据合并模块
```

### 3.4 核心类

| 类名 | 说明 |
|------|------|
| `KLineDataLoader` | K线数据加载器，负责读取 Excel 格式的 K线数据 |
| `DataMerger` | 数据合并器，支持多种合并方式 |
| `DataExporter` | 数据导出器，支持导出 Excel/CSV 格式 |

### 3.5 使用示例

```python
from src.data import KLineDataLoader, DataMerger, load_kline

# 方式一：使用便捷函数
df = load_kline("000001", "daily")

# 方式二：使用加载器类
loader = KLineDataLoader()
df = loader.load_kline("000001", "daily")

# 加载多只股票
symbols = ["000001", "600000", "000002"]
df = loader.load_multiple(symbols, period="daily", merge=True)

# 使用合并器
merger = DataMerger()

# 按日期合并（横向，用于比较分析）
df = merger.merge_by_date(symbols, period="daily")
# 结果列: 000001_open, 000001_close, 600000_open, 600000_close, ...

# 按股票合并（纵向，用于批量分析）
df = merger.merge_by_symbol(symbols, period="daily")
# 结果列: symbol, date, open, close, ...

# 合并不同周期
df = merger.merge_periods("000001", periods=["daily", "min30"])
# 结果列: open_daily, close_daily, open_min30, close_min30, ...
```

| 成交额/Amount | amount |
