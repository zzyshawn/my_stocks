# 量化投研模块需求文档 (Quant Module)

## 模块概述
本模块是在基础数据与分析模块之上全新拓展的工业级 AI 量化投研框架深度集成平台。该模块不再使用分散独立的分析逻辑，而是完整拥抱微软开源的 `QLib` 原生架构，提供一套高内聚、标准化的“数据整理 -> AI 训练 -> 回测量化”全闭环流水线。

## 核心设计要求
1. **彻底的动态配置无硬编码**：必须依赖 `config.yaml` 作为全局路径和时间的唯一源解析，适应各种电脑与网络映射。
2. **轻量衔接**：不干扰系统目前原生的 Excel 获取数据与基础 K 线逻辑，仅在此基础上桥接产生 QLib 的环境。
3. **原生兼容性**：核心分析层不能自造轮子，而应该拥抱诸如 `Alpha158` 等行业通用量化表达算子，实现代码的标准化极简。

## 功能需求详述

### 1. 本地数据 -> 量化数据管道 (`src/data/qlib_dump.py`)
- **[目标]** 支持将本地任意 Excel K线格式的数据表转化为 QLib 能极速载入和寻址的 `.bin` 高性能文件库。
- **[能力]** 结合 `backtest.target_list` 股票池读取，实现多线程多目标个股的数据规整（包含自动注入 `date` 与 `symbol` 参数并完成格式安全校验），全量输出至配置指定的 `data.qlib_dir` 目录。
- **[状态]** ✅ 已开发完成使用。

### 2. 因子特征提纯工厂 (`src/analysis/factors.py`)
- **[目标]** 在 `config.yaml` 动态分配的中心化数据库上提取任意因子系列。
- **[能力]** 封装 `QlibFactorEngine`，支持动态从指定目录 `qlib.init` 初始化项目域。利用原生的 `Alpha158` 获取百种经典特征矩阵。通过 `DataHandlerLP` 让使用者能书写诸如 `Ref($close, 1)` 或多周期聚合 `Resample` 之类的原生算子，将计算过程彻底交给 C++ 底层加速。
- **[状态]** ✅ 已开发完成使用。

### 3. 量化机器学习底座 (`src/models/train_lgb.py`)
- **[目标]** 构建标准的数据投喂与模型预测流程。
- **[能力]** 基于 `DatasetH` 自动化从 `config.yaml` 切分 Train / Valid / Test 动态数据集时间段。完成包含：
  - Label 衍生生成：采用 `Ref($open, -5) / $open - 1` 指定未来五日收益为优化指标。
  - 数据清洗处理：接入 `RobustZScoreNorm` 执行极值正态缩放与 `Fillna` 空值填充清洗流。
  - 模型训练推理：挂载 LightGBM（支持自定义超参数配置树深度与叶子约束），一键实现训练并在 Test 数据切片上推理生成预测真实信号（Score Series）。
- **[状态]** ✅ 已跑通模型全流。

### 4. 基于 AI 的全市场回测执行器 (`src/backtest/strategies/qlib_demo.py`)
- **[目标]** 提供对未见 Test 数据打分信号的全方位风控评估。
- **[能力]** 把底层从手写循环替换为 `qlib.backtest.executor.SimulatorExecutor`。配置每日买开盘/收盘频率映射。承运 `TopkDropoutStrategy` 对 `train_lgb.py` 的 AI 得分信号执行自动多/空调仓操作。控制台呈现并记录资金曲线、夏普比率、最大回撤等回测验证参数。
- **[状态]** ✅ 构建完备与验证通过。
